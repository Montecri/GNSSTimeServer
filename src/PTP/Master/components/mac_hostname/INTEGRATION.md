# mac_hostname — deterministic, fully reversible MAC → hostname

Turns the device's Ethernet MAC into a stable, human-friendly hostname
(`prefix-title-first-last`, e.g. `fabled-roper-susanda-perrhale`) and
back. The mapping is **bijective over the entire 2^48 MAC space**:
every MAC yields a distinct hostname, and any generated hostname
decodes to exactly its MAC — no collisions, no lookup registry.

## Layout in the project

```
your-project/
├── main/
│   └── main.c                     <- one buffer-size edit, see below
└── components/
    └── mac_hostname/
        ├── CMakeLists.txt
        ├── include/
        │   └── mac_hostname.h     <- public API
        ├── mac_hostname.c         <- codec (Feistel + encode + decode)
        ├── mac_hostname_esp.c     <- mac_hostname_get / mac_hostname_apply
        ├── name_data.c            <- AUTOGENERATED tables (~10 KB flash)
        ├── name_data.h            <- AUTOGENERATED struct + externs
        ├── INTEGRATION.md         <- this file
        └── tools/                 <- not compiled into the firmware
            ├── name_components.py <- curated word/stem/ending lists
            ├── emit_c_data.py     <- regenerates name_data.c/h
            ├── sample_names.py    <- PC codec + full self-test
            ├── emit_c_data.json   <- table dump used by sample_names.py
            └── host_selftest.c    <- PC test of the real C sources
```

No top-level CMakeLists.txt change is needed: ESP-IDF auto-discovers
`components/`, and the `main` component automatically depends on every
discovered component. (If your `main/CMakeLists.txt` lists explicit
`PRIV_REQUIRES`, add `mac_hostname` there.)

## Required edit in main.c

The v3 scheme's worst-case "first-last" OLED line is 17 chars + NUL:

```c
static char g_host_line2[16];   /* before */
static char g_host_line2[24];   /* after  */
```

`g_host_line1[24]` and `g_hostname[MAC_HOSTNAME_BUF_LEN]` are already
correctly sized. Nothing else in main.c changes —
`mac_hostname_get()`, `mac_hostname_apply()` and
`mac_hostname_display_lines()` keep their signatures.

Kconfig: `CONFIG_LWIP_MAX_HOSTNAME_LEN` must be >= 32 (hostnames are
15..31 chars).

## How it works

`MAC --feistel_forward--> 48-bit index --split--> 8|7|17|16 bits`

| bits | token  | construction                                   |
|------|--------|------------------------------------------------|
| 8    | prefix | 256 real adjectives (≤ 6 ch)                   |
| 7    | title  | 128 real English roles/ranks (≤ 6 ch)          |
| 17   | first  | 256 stems × 512 endings (3–8 ch)               |
| 16   | last   | 256 roots × 256 suffixes (3–8 ch)              |

First/last names are composed, not stored: the stem tables are
**prefix-free** (no stem is a prefix of another), which makes
`stem+ending` concatenation provably injective and lets the decoder
recover the split with a deterministic prefix scan. Decoding is
case-insensitive. Worst-case dashed length is 31 (+ NUL = 32).

## Regenerating or tuning the tables

The component lists in `tools/name_components.py` are plain curated
lists — edit to taste, then:

```
cd components/mac_hostname/tools
python3 emit_c_data.py          # re-proves all invariants, writes
cp name_data.c name_data.h ..   # tables next to the sources
python3 sample_names.py test    # full self-test (bijectivity, 100k)
```

`emit_c_data.py` machine-checks every invariant before writing
anything: exact table sizes, prefix-freeness, exhaustive uniqueness of
all 2^17 + 2^16 concatenations, the 31-char worst case and the 3-char
minimum. If an edit breaks a proof, it refuses to emit.

Note: editing the tables (or the Feistel keys in `emit_c_data.py`)
changes every device's hostname on the next flash.

## PC-side verification

```
cd components/mac_hostname
gcc -Wall -Wextra -O2 -Iinclude -I. -o /tmp/mh_test \
    mac_hostname.c name_data.c tools/host_selftest.c && /tmp/mh_test
```

Runs the real firmware sources on the host: sample MACs, round-trips,
case-insensitive decode, display-line buffer checks, and 100 000
random MACs verified unique and reversible. `sample_names.py` is the
independent Python implementation; both produce identical output
(e.g. `dc:1e:d5:f8:b2:d8 → dulcet-keeper-vicant-pecklund`), so you can
decode a hostname seen in DHCP/mDNS back to its MAC on a PC with
`sample_names.py`, or on-device with `mac_hostname_to_mac()`.
